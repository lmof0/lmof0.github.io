array notation:
# refers to the unchanged remainder of the array.
base rule: [a,b] = ab
prime rule: [a,1,#] = a
tailing rule: [#,1] = [#]
hyperoperation rule: [a,b,1,...,1,c,d,#] = [a,a,...,[a,b-1,1,...,1,c,d,#],c-1,d,#]
recursion rule: [a,b,c,#] = [a,[a,b-1,c,#],c-1,#]

or in english,

if there are two entries, the array equals the first entry to the power of the second.
if the second entry is 1, the array equals the value of the first entry.
if the last entry is 1, it can be pruned.
if the third entry is one then:
- all elements before the next non-1 element become the first element.
- the last 1 in the string of 1s becomes the original array with the second entry decreased by 1.
- the said non-1 element is decreased by one.
if the above rules don't apply, let the second entry equal the entire array but with the second entry in that new array decreased by one, and subtract one from the third entry in the original array.

the above should operate exactly as bowers's original array notation from 2002. (his original array notation just lines up so well with how one would naturally extend the hyperoperators, so i'm using those rules)

according to these rules, the limit of this notation is about ωω on the FGH.

extended array notation:
unextension rule: [#](1) = [#]
extended base rule: [a,b](n) = [a,a,a,a,...,a](n-1) with b as
extended prime rule: [a,1,#](n) = a
extended tailing rule: [#,1](n) = [#](n)
extended hyperoperation rule: [a,b,1,...,1,c,d,#](n) = [a,a,...,[a,b-1,1,...,1,c,d,#](n),c-1,d,#](n)
extended recursion rule: [a,b,c,#](n) = [a,[a,b-1,c,#](n),c-1,#](n)

examples:
[10,10](2) = [10,10,10,10,10,10,10,10,10,10]
[10,100](2) = [10,10,10,10,...,10,10,10,10] where there are 100 tens
[10,3,2](2) =
[10,[10,2,2](2),1](2) =
[10,[10,[10,1,2](2),1](2),1](2) =
[10,[10,10](2)](2) =
[10,[10,10,10,10,10,10,10,10,10,10](1)](2) =
[10,10,10,10,................,10,10,10,10] where there are [10,10,10,10,10,10,10,10,10,10] tens.

the limit of [a,b,2](2) on the FGH is ωω+1.
the limit of [a,b,c](2) on the FGH is ωω+ω.
the limit of [a,b,c,d](2) on the FGH is likely ωω+ω2.
the limit of [a,b,c,d,e](2) on the FGH is likely ωω+ω3.
the limit of [a,b](3) on the FGH is likely ωω+ωω = (ωω)2.
the limit of [a,b](4) on the FGH is likely (ωω)3.
the limit of [a,b](n) on the FGH is likely (ωω)ω = ωω+1.
the next step would be to find an extended expression using similar rules [a,b](n,2)? which on the FGH would be equivalent to (ωω)ω2 = ωω+2.


definitions for groups and gaggles of googologisms:
group: family size of less than 500 members
gaggle: family size of between 500 and 1,000 members
regiment: family size of between 1,000 and 5,000 members
super regiment: family size of between 5,000 and 10,000 members
gathering: family size of 10,000 members or more
grand googolo gathering: family size of 1010 members or more
