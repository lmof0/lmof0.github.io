instead of asking what a pizza is, ask what the simplest pizza is.

i argue that cheese pizza is the simplest, and that all pizzas can be constructed from cheese pizza.

so the set of all pizzas contains the simplest pizza plus any transformation/topping (this can be a set because pizzas aren't sets, so there will be no russell's paradox).

it is crucial that this transformation/topping is always additive. if you assume you could subtract, then you would be able to get a simpler "pizza" than the simplest pizza itself. this is a logical contradiction.

formally,

∃p∀t((p + t) ∈ P)
P = { x : x = (p + t) } (this property totally defines P)

where t is a given transformation/topping,
p is the simple pizza,
and P is the set of all pizzas.

~~~~~~~~~~~~~~~~~~~~~~

one can argue that cheese pizza is not the simplest, and that there may be more than one root pizza (especially when considering regional differences)
a pizza is the sum of its parts, so we can't just keep simplifying our simple pizza. it's already nearly too simple now.
so we define S, the set of all simple pizzas, which will provide all bases for which to construct every pizza. now:

∃S∀p(p ∈ S)∀t[((p + t) ∈ P) ∧ ((t ≠ ∅) → ((p + t) ∉ S))]
(i am unsure how to formally notate this)

where t is the transformation/topping,
p is a given simple pizza,
S is the set of all simple pizzas,
and P is the set of all pizzas,

the set P is a superset of S.

and now we can construct a much wider variety of pizzas without resorting to a single overly simple starting point.

~~~~~~~~~~~~~~~~~~~~~~

my hope is that this thesis can finally put an end to semantic debates over what makes a pizza a pizza.
